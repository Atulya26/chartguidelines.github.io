01

Categorical comparison

Bar Chart

Compare named categories on a shared scale.

Overview Variants Questions Mistakes

BarChart

What this chart is for

Use a bar chart when the reader needs to compare items, rank categories, or scan differences quickly. It is the safest default for categorical comparison because length on a common baseline is easy to judge.

How readers decode it

Readers compare bar length first, then category labels, then exact values. Order and baseline matter more than decorative styling.

Use when

  • Rank categories from high to low
  • Compare teams, regions, products, cohorts, or modules
  • Show grouped values across the same categories
  • Show total plus composition with stacked bars

Avoid when

  • Time-series movement where shape matters
  • Part-to-whole stories with only a few pieces
  • More than 6 visible category colors
  • Non-zero baselines that visually exaggerate small differences

Variants

Best default for one metric across categories.
Sort intentionally: highest first, lowest first, or product-defined order.
Grouped bars

Avoid beyond three series; scanning becomes slow.
Stacked bars
Use when the total and broad composition both matter.
Only the first segment has a shared baseline. Do not use it for precise segment comparison.
Horizontal bars
Use for long category names or rankings.
Prefer this before rotating labels.
Distribution bars

  3. How many series are truly needed?
  4. Do exact values need to be visible or can they live in hover?

Natural product use cases

  • Top regions by completed tasks
  • Plan comparison by member count
  • Tickets by priority
  • Product modules ranked by adoption

02

Trend over time

Line Chart

Show movement across ordered points.

Overview Variants Questions Mistakes

LineChart

What this chart is for

Use a line chart when the important thing is direction, slope, volatility, or change over time. The shape of the line should be the main message.

How readers decode it

Readers read the overall shape before individual values. Gaps, slopes, peaks, and thresholds should be honest and easy to inspect.

Use when

  • Time-series trends
  • Actual versus projected movement
  • Small multi-series trend comparisons
  • Targets, thresholds, baselines, or averages

Avoid when

  • Unordered categories
  • More than four competing lines
  • Connecting across missing time gaps without explanation
  • Dual axes without very clear labeling

Variants

03

Bars plus trend

Combo Chart

Combine quantity and related context.

Overview Variants Questions Mistakes

ComboChart

What this chart is for

Use a combo chart when bars and a line answer one related question together: volume plus rate, actuals plus target, or totals plus benchmark.

How readers decode it

Readers read the bars as the base quantity and the line as the contextual layer. The relationship must be intentional, otherwise the chart invents a story.

Use when

  • Volume plus rate
  • Actual totals plus target trend
  • Category totals with a benchmark line
  • Two related metrics with different scales

Avoid when

  • Unrelated metrics
  • More than one overlay line
  • Area fills over bars
  • Too many grouped bar series under the line

Variants

Do not present this as a separate chart type.
Dual-axis combo

04

Part of whole

Donut Chart

Explain composition of one whole.

Overview Variants Questions Mistakes

DonutChart

What this chart is for

Use a donut chart when the story is composition. It works best with a small number of segments and when approximate share is enough.

How readers decode it

Readers read the dominant segment, then the center value or legend. Donuts are weak for precise comparisons, so the story must be broad composition.

Use when

  • Share of a whole
  • One dominant segment
  • Compact composition cards
  • A center label with total or selected value

Avoid when

  • Precise comparison between many segments
  • Time-series data
  • More than five or six slices
  • Tiny segments that cannot be labeled or selected

What this chart is for

Use a half donut when the story is one progress, score, or completion value. It is compact and expressive, but not good for comparing many values.

How readers decode it

Readers read the center value first, then where the arc sits between min and max. The range must be meaningful.

Use when

  • Score summaries
  • Goal progress in compact cards
  • One value against a bounded range

Avoid when

  • Unbounded values

Use a histogram when the story is spread, clustering, skew, or outliers across numeric ranges. The bin logic is the design decision.

How readers decode it

Readers read the overall distribution shape first, then bin labels. If bins are unclear, the chart becomes misleading even if it looks polished.

Use when

  • Frequency distribution
  • Understanding spread and clustering
  • Showing skew or outliers
  • Comparing ranges rather than named categories

Avoid when

  • Overlapping or uneven bins without explanation
  • Too many narrow bins
  • Using it as a normal bar chart

  • Do not rely on color alone for meaning.
  • Write a useful title and description for handoff.
  • Keep the chart understandable before hover.
  • Use table fallback where precision matters.

State guidance

If history is missing, show the current value without a sparkline rather than drawing a flat fake trend.

Shared color logic

Use tokens, then add meaning

The library uses categorical slots, sequential ramps, sentiment colors, pattern fills, and accessible outlines. Designers should choose intent first: category, order, status, or interaction.

Category

Separate unrelated groups or series. Keep the number of visible colors low enough to remember.

Sequence

Show ordered magnitude or range intensity. Pair color with labels or axes.

Status

Use sentiment color only when product meaning is explicit and never as the only cue.

Implementation handoff

Keep handoff useful, but keep guidance designer-first

The handoff should document chart family, story intent, data shape, labels, states, and accessibility copy. For deeper technical background, use the repo and Storybook.
